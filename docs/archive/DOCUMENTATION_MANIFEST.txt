================================================================================
KEY TAKEAWAYS
================================================================================

1. ARCHITECTURE IS SIMPLE
   - 3 dependencies (Astro, Svelte, @astrojs/svelte)
   - Clear separation: tokens → widgets → sections → pages
   - No hidden magic or runtime discovery

2. WIDGET INDEPENDENCE
   - Each widget defines its own parameters
   - Widgets never reference global spacing tokens
   - Per-widget debug panels for real-time tuning

3. ANIMATION PATTERNS
   - Interval-based grid mutation (DOM refs + class toggling)
   - SVG stroke-dash animations (getTotalLength + CSS transitions)
   - Canvas continuous animation (frame counter + setInterval)
   - All have visibility observers + document.hidden checks

4. DEBUG SYSTEM
   - Global panel: design tokens (bottom-right, Ctrl+.)
   - Per-widget panel: widget params (gear icon)
   - Two layers completely independent
   - Both removed from production via import.meta.env.DEV

5. DEVELOPMENT WORKFLOW
   - Create widget in isolation (sandbox page)
   - Tune with debug panel in real-time
   - Integrate into section with prose
   - Add to essay page

================================================================================
IMPORTANT: THESE ARE RULES
================================================================================

From CLAUDE.md - These OVERRIDE all other patterns:

1. Widgets must NOT reference global spatial tokens
   (--space-*, --radius-*, layout widths)

2. All widget numeric styling comes from widget's own paramDefs

3. Non-spatial globals are fine
   (--color-*, --font-*, --transition-*)

4. Widgets include <WidgetDebugPanel> unconditionally

5. Style params flow via scoped CSS custom properties

6. Behavioral params used directly in JS

7. Sections are Svelte components (not Markdown)

8. Every section's <h2> needs unique id for TOC
