## KNOWN_TENSIONS
- GIF demos vs. static screenshots: GIFs can be distracting, large, slow to load — but they show interactivity better than screenshots
- Long comprehensive README vs. short focused README linking out to docs — community divided
- "Install & run" instructions for developers vs. "try it live" link for casual evaluators — different audiences need different hooks
- Badges (CI status, npm version, license) — some argue essential for credibility, others say visual noise

## LIKELY_GAPS
- SvelteKit-specific README conventions may have thin coverage (relatively newer ecosystem)
- Research on what works for CS education tool README specifically (very niche)
- Empirical data on README effectiveness (most guidance is opinion-based)
- Academic/CS education community expectations vs. general open-source expectations

## RECENCY_SENSITIVITY
Medium-high. README practices evolve with platform features (GitHub now supports video embeds, improved image rendering). Bias toward 2024-2026 sources. Flag anything older than 2022.
