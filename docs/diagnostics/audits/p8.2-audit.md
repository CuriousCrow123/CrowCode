## Bugs Found
- [ ] BUG-p8.2-1: Variable shadowing is not modeled correctly. The inner `int x = 20` (line 4) overwrites main::x via setValue instead of creating a separate scoped entry. The user sees x change from 10 to 20, then to 25, with no indication of shadowing. The inner block scope is completely missing.
- [ ] BUG-p8.2-2: At snapshot 4 (after inner block exits), main::x still shows val=25. The outer x should have been restored to 10. However, the actual runtime correctly passes x=10 to `int y = x` (y gets val=10), so the WASM runtime distinguishes the two variables internally. The display is wrong but the underlying computation is correct.
- [ ] BUG-p8.2-3: No inner scope is pushed or popped for the `{ }` block. The instrumented source lacks `__crow_push_scope`/`__crow_pop_scope` for the inner block, so shadowing cannot be represented.
- [ ] BUG-p8.2-4: Cleanup step reports Line 7 instead of Line 8.
