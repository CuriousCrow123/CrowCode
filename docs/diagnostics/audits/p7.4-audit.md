## Bugs Found

- [ ] BUG-p7.4-1: **Redundant intermediate setValue ops on for-loop increment.** Same pattern as p7.3. Every iteration where the body completes normally produces two setValue ops for i: one pre-increment (from explicit `__crow_set`), one post-increment (auto-detected at step boundary). The pre-increment value is stale. Occurs at Steps 4, 6, 8, 11, 13, 15. Final snapshot values are always correct.

- [ ] BUG-p7.4-2: **Continue step is invisible.** When i=3 and `continue` fires, the step for the continue line (source line 4) is empty (no ops between step(3) and step(4)) and is omitted from the dump. The user jumps from Step 8 (i becomes 3) directly to Step 9 (i becomes 4) with no indication that `continue` was hit. The break at i=7 IS visible (Step 16, Line 5) because it coincides with the removeEntry op, but continue has no ops and is invisible.

- [ ] BUG-p7.4-3: **Break step conflates break with cleanup.** Step 16 (Line 5) contains only removeEntry(main) from `__crow_pop_scope()`. The break itself generates no dedicated op. While the line number (5) correctly points to the `break` line, the only visible effect is scope removal. There is no step showing the user that line 5 was reached and the `if (i == 7)` condition was true -- the condition check step (between step(3) and step(5)) is empty and omitted.
