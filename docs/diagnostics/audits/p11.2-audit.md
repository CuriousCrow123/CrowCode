## Bugs Found

- [ ] BUG-p11.2-1: `matrix[i*rows+i] = 1` writes through a pointer to heap memory, but the instrumented code emits `__crow_set("matrix", &matrix, 6)` which reports the pointer variable's own value (0x00003038) rather than the heap data. The heap entry `heap_0` never reflects the written values. The user cannot see that the diagonal elements were set to 1. This is the same dereference-write problem seen in p14.1 and p5.3.
- [ ] BUG-p11.2-2: calloc is labeled `malloc(36)` in the heap entry name. The source uses `calloc(rows * rows, sizeof(int))` but the heap entry shows `name=malloc(36)`. Should show `calloc(36)` or `calloc(9, 4)` to distinguish zero-initialized allocation from uninitialized allocation.
