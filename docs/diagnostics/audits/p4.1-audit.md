## Bugs Found
- [ ] BUG-p4.1-1: When writing through a pointer (`*p = 42`), the heap entry's value is never updated. The setValue op only updates the pointer variable `p` (which still holds the same address). The dereferenced value (42, then 50) is invisible to the user.
- [ ] BUG-p4.1-2: The `__crow_set("p", &p, ...)` instrumentation records the pointer's value, not the pointed-to value. For `*p = 42`, the system should update heap_0's val to 42, but it only re-records p's address value.
- [ ] BUG-p4.1-3: Cleanup step (Step 5) reports Line 6 instead of Line 7 (`return 0`). Same instrumentation pattern as struct programs.
