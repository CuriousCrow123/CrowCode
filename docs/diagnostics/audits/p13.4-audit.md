### Value verification details:
- `int x;` is uninitialized. WASM zero-initializes memory, so showing 0 is technically the runtime value. However, for a memory visualizer, it would be more educational to mark this as "uninitialized" or "?" rather than showing 0, since C does not guarantee 0 for uninitialized local variables. This is a known limitation of running on WASM.
- `y + 5`: 10 + 5 = 15. Correct.
- `x * 2`: 15 * 2 = 30. Correct.

## Bugs Found

- [ ] BUG-p13.4-1: Uninitialized variable `x` shows val=0 instead of being marked as uninitialized. WASM zero-initializes stack memory, so the runtime value is technically 0, but this obscures the fact that reading an uninitialized variable is undefined behavior in C. A memory visualizer should ideally flag this.
