## Bugs Found

- [ ] BUG-p5.3-1: `struct Point origin` has no visible children. Step 1 adds origin with `children=0`. Expected children `x=0` and `y=0` to be visible as sub-entries. The struct is declared but its fields are not exposed.
- [ ] BUG-p5.3-2: Arrow assignments (`p->id`, `p->pos.x`, `p->pos.y`, `p->scores`) all emit `setValue(main::p, val=0x00003090)` which just re-reports the pointer value. The heap entry `heap_0` never shows the fields `id=1`, `pos.x=3`, `pos.y=4`, or `scores` pointer. The user cannot see the internal state of the heap-allocated struct.
- [ ] BUG-p5.3-3: `p->scores[0]=100`, `p->scores[1]=200`, `p->scores[2]=300` (lines 16-18) produce no visible steps for those values. The instrumented source uses `__crow_set("p->scores", &p->scores, ...)` which reports the pointer value of `p->scores`, not the array data. Steps for lines 16-18 are entirely absent from the dump -- the program jumps from step 6 (line 15, calloc) directly to step 7 (line 4, distance function entry).
- [ ] BUG-p5.3-4: Struct parameters `a` and `b` in the `distance` function have no children visible. `distance::a` and `distance::b` are type `struct Point` with no sub-entries for `.x` and `.y`, so the user cannot see the passed values `(0,0)` and `(3,4)`.
