## Bugs Found

- [ ] BUG-p16.5-1: `scanf("%d", &num)` on line 5 reads input value 42 but produces no setValue op. Snapshot after step 3 still shows num=0. Same root cause as BUG-p16.4-1 -- `__crow_scanf_int` does not emit a setValue op.
- [ ] BUG-p16.5-2: `scanf("%c", &ch)` on line 6 reads the newline residue character but produces no setValue op. Snapshot after step 4 still shows ch=0. The `__crow_scanf_char` wrapper also does not emit a setValue op.
- [ ] BUG-p16.5-3: The IO event for the `scanf("%c", &ch)` read appears to show an empty value (`IO: read` with no visible character). This is technically correct (the newline character '\n' was consumed from the buffer after "42\n"), but the display could be clearer -- it should show the character that was read (e.g., '\n' or its numeric value 10).
