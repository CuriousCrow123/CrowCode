## Bugs Found

- [ ] BUG-p16.3-1: Step 4 (cleanup/return) is reported at Line 5 instead of Line 10. The `return 0;` is on source line 10, and the `__crow_pop_scope()` occurs before it. The instrumented source doesn't have an explicit `__crow_step` between pop_scope and return, so the line number carries forward from the last `__crow_step(5)` call. However, the dump shows the cleanup step at Line 5 which is the line for `c = getchar()`, not for `return 0;`. This is misleading for the user since the cleanup appears to happen on the getchar line rather than near the return statement.
- [ ] BUG-p16.3-2: No IO event is recorded for the `getchar()` call on line 5. Even though getchar returns EOF (no stdin provided), the dump should ideally show an IO read event. Instead, only the setValue op is recorded. Compare with p16.4 where scanf shows IO: read events.
