## Bugs Found

- [ ] BUG-p12.1-1: Inner loop variable `j` is not incremented to its exit value on loop termination. At each inner loop exit, j should be incremented one more time (the increment that causes the condition to fail), but the `__crow_set("j", &j, 4)` at end of loop body captures the pre-increment value. This occurs at Steps 19, 29, 35, and 37. Same root cause as BUG-p3.1-1.
- [ ] BUG-p12.1-2: Outer loop variable `i` is not incremented to its exit value (4) on final loop termination at Step 37. The dump shows i=3 in the removeEntry step, but i should be 4 (the value that causes `i < 4` to fail). Same root cause.
- [ ] BUG-p12.1-3: Step 37 line attribution is Line 4 (inner for) but this step also includes the outer loop exit and scope cleanup (return 0). The line should arguably be Line 3 or Line 12.
