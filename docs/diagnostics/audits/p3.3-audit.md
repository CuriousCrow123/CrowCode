### Value verification details:
- data[0]*data[0] = 1*1 = 1. Correct.
- data[1]*data[1] = 2*2 = 4. Correct.
- data[2]*data[2] = 3*3 = 9. Correct.
- data[3]*data[3] = 4*4 = 16. Correct.
- total accumulation: 0+1=1, +4=5, +9=14, +16=30. All correct.
- Note: The two `for` loops reuse the same `i` variable (same id `main::i`). The second loop's `__crow_decl` does a setValue(i,0) rather than creating a new entry, since `main::i` already exists. This is an instrumentation quirk but the values are correct.

**Step 10 issue:** Same pattern as p3.1. The first loop's final i++ (from 3 to 4) is not captured. The `__crow_set("i", &i, 3)` at end of loop body emits the pre-increment value (3), then the loop exits. The increment to 4 is never recorded.

**Step 19 issue:** Same bug for the second loop. Final i++ from 3 to 4 is not captured.

Also note: Step 10 and Step 19 have slightly wrong line attributions. Step 10 should relate to the for-exit/line 6 declaration, and Step 19 should be the return statement line, not line 8.

## Bugs Found

- [ ] BUG-p3.3-1: First for-loop exit: final i++ not reflected. After last iteration (i=3), the loop should increment i to 4 before condition check fails, but dump shows i=3 at step 10. Same root cause as BUG-p3.1-1.
- [ ] BUG-p3.3-2: Second for-loop exit: final i++ not reflected. After last iteration (i=3), the loop should increment i to 4, but dump shows i=3 at step 19. Same root cause.
- [ ] BUG-p3.3-3: Step 19 is attributed to Line 8 (loop body) but should be the loop exit / return statement.
