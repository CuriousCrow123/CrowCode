## Bugs Found

- [ ] BUG-p16.4-1: `scanf("%d", &x)` on line 6 reads input value 10 (IO: read 10) but produces no setValue op. The snapshot after step 4 still shows x=0. The WASM backend correctly executes the scanf (the printf on line 8 outputs "Sum = 30" proving x=10 and y=20 internally), but the instrumentation fails to emit a setValue op to update the visible memory entry. The `__crow_scanf_int` wrapper should emit a setValue for the target variable after reading the value.
- [ ] BUG-p16.4-2: `scanf("%d", &y)` on line 7 has the same problem -- reads input value 20 (IO: read 20) but no setValue op is emitted. Snapshot after step 5 still shows y=0. Same root cause as BUG-p16.4-1.
