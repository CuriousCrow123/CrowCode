## Bugs Found
- [ ] BUG-p4.4-1: Heap array element writes (`squares[i] = i*i`) never update the heap entry. All setValue ops only re-record the pointer address. The computed squares values (0, 1, 4, 9, 16) are invisible.
- [ ] BUG-p4.4-2: Loop increment steps emit a redundant setValue before the actual increment (e.g., Step 5 sets i=0 then i=1). The first setValue is the stale pre-increment value. Harmless for final state but wasteful.
- [ ] BUG-p4.4-3: At loop exit (Step 13), i is set to val=4 but should be 5 (the value that caused `i < n` to fail). The final `i++` from 4 to 5 is not recorded.
- [ ] BUG-p4.4-4: Cleanup step (Step 14) reports Line 8 instead of Line 9.
