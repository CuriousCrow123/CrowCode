## Bugs Found

- [ ] BUG-p13.7-1: 2D array type is `int[3]` instead of `int[3][3]`. The instrumenter emits `__crow_decl("m", &m, sizeof(m), "int[3]", 2)` losing the second dimension. The type should be `int[3][3]`.
- [ ] BUG-p13.7-2: 2D array children are flat. Snapshot 1 shows `m` with 3 children `[0]=1, [1]=0, [2]=0` -- these are the first 3 ints of the flattened array, not 3 sub-arrays each containing 3 ints. A 3x3 identity matrix should show `[0]` as `{1,0,0}`, `[1]` as `{0,1,0}`, `[2]` as `{0,0,1}`. The children addresses (0x1fd0, 0x1fdc, 0x1fe8) suggest 12-byte spacing (3 ints), so the grouping is correct at the row level, but values only show the first element of each row.
- [ ] BUG-p13.7-3: `m[1][2] = 5` assignment (line 7) produces no setValue op for `m`. Step 9 only emits `setValue(i, 2)` (leftover loop increment). The instrumented source has `__crow_set("m[1]", &m[1], 7)` but it appears to land on step 9 which only shows `i` being set. Similarly `m[2][0] = 7` on line 8 produces no visible update -- step 10 immediately removes main.
- [ ] BUG-p13.7-4: Steps 9 and 10 are misaligned. Step 9 is labeled Line 7 but only sets `i=2` (loop cleanup). Step 10 is labeled Line 8 but only does `removeEntry(main)`. The `__crow_step(7)` and `__crow_step(8)` that should show the m[1] and m[2] updates appear to be missing or merged with the loop exit and scope pop steps.
