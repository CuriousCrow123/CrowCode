## Bugs Found

- [ ] BUG-p16.7-1: `scanf("%d", &score)` never produces a setValue op for `main::score`. This occurs on all 4 scanf calls (steps 6, 11, 16, 20). The `score` variable remains at val=0 in every snapshot throughout the entire program. Despite this, the program computes correct results internally (total, count, highest, printf outputs are all correct), proving the WASM backend reads the value correctly but the `__crow_scanf_int` wrapper does not emit a setValue op to the trace. This is the same root cause as BUG-p16.4-1 and BUG-p16.5-1.
