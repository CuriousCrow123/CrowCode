## Bugs Found

- [ ] BUG-p9.1-1: All 6 sprintf calls (lines 5-10) produce no visible steps. The instrumented source contains `__crow_step(5)` through `__crow_step(10)` but only 4 total steps are recorded (0, 1, 2, 3). Steps for lines 5-10 are completely missing from the dump. Since sprintf writes to heap memory via a pointer and no `__crow_set` is emitted for any of these calls, the steps produce zero ops and appear to be collapsed/omitted. This means the user sees the program jump from `malloc` directly to `free` with no indication that 6 sprintf calls occurred in between.
