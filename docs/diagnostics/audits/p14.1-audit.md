## Bugs Found

- [ ] BUG-p14.1-1: `*p = 42` (line 4) does not update the heap entry's value. Step 2 emits `setValue(main::p, val=0x00003028)` which just re-sets the pointer value (unchanged). The instrumented source has `__crow_set("p", &p, 4)` -- this reports `p`'s own value (the pointer address), not the dereferenced value. The heap_0 entry never shows val=42. The instrumenter should emit a dereference-aware set that updates the heap entry's displayed value.
- [ ] BUG-p14.1-2: Use-after-free on line 6 (`int x = *p` after `free(p)`) is not detected or flagged. The program reads from freed memory and gets 42 (stale data still in WASM linear memory). There is no warning, no error op, and no visual indication that this is a use-after-free. The value x=42 is silently accepted.
