## Bugs Found
- [ ] BUG-p15.1-1: All writes through `player->` pointer (id, pos.x, pos.y, score, and later score += 10) produce only setValue on the pointer variable, never updating the heap entry. The heap block has no children and val remains empty throughout. Combines heap-deref and struct-field bugs.
- [ ] BUG-p15.1-2: Stack-local struct `dir` (struct Vec2) has val= and children=0. Its fields {1, 0} are never visible. Same as p2.1 struct bug.
- [ ] BUG-p15.1-3: At Step 13 (function return), the loop variable i is set to val=0 (stale) instead of val=1 (the value that caused loop exit). Same loop-exit bug as p4.4/p11.5.
- [ ] BUG-p15.1-4: Cleanup step (Step 16) reports Line 24 instead of Line 25.
- [ ] BUG-p15.1-5: The `if (total > 50)` condition evaluation has no dedicated step. The user jumps from Step 13 (total=100 declared) directly to Step 14 (inside if body). This is a minor UX issue -- the conditional check itself is invisible.
- Note: Function call/return for sumScores works correctly -- scope push, parameter passing, local variables, return value propagation back to main::total all function properly. Cross-function flow is sound.
