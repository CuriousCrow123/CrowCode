### Step 7 line number note

Step 7 is tagged Line 7, but the `return 0;` statement is on source line 8. The instrumented code places `__crow_pop_scope()` after `__crow_step(7)` (which corresponds to line 7), so the main scope cleanup is recorded under the step for line 7 rather than line 8. This is because the pop happens between step(7) and what would be step(8), and the ops are flushed with the next step boundary. The `return 0` on line 8 has no dedicated step.
